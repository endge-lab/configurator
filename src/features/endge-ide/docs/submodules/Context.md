# Endge.context

Модуль `Endge.context` хранит runtime-контекст ядра: текущий проект, среду и локаль.

Основная роль:
- единая точка доступа к текущему проекту;
- хранение выбранной среды исполнения;
- хранение локали, используемой UI-адаптерами;
- сериализация контекста без смешивания с состоянием boot pipeline.
