# Основные понятия NOVA

## NovaGraph

Фазовый граф, который управляет порядком и областью обновлений:

- `preupdate`
- `update`
- `matrix`
- `render`
- `flush`

Обновляются только `dirty`-узлы, поэтому рендер предсказуем и экономный.

## NovaApp

Корневой объект приложения:

- инициализирует canvas и рендер;
- хранит `store` и event-bus;
- управляет поверхностями и lifecycle (`startLoop`, `stopLoop`, `resize`, `destroy`).

## NovaSurface

Отдельный слой отрисовки.

У каждого surface свой canvas/renderer. На `flush` слои композятся в основной canvas.

## NovaNode

Базовый объект сцены с:

- геометрией (`x`, `y`, `width`, `height`);
- матрицей трансформации;
- видимостью/активностью;
- обработчиками интерактивности.

## Рендереры

- `NovaRenderer2D` - полноценный 2D путь;
- `NovaRendererWebGL` - производительный путь (часть API может быть ограничена в текущей версии).

## Practical note

Хорошая стратегия: использовать `@endge/raph` для данных и подписок, а в NOVA менять состояние узлов и вызывать `invalidate()`.
