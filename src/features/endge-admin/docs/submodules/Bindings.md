# Endge.bindings

`Endge.bindings` - это модуль declarative bindings.

Он не подменяет код сущностей напрямую, а работает с конфигурацией реакций:

- резолвит итоговые `binding` по `owner / target / event`;
- применяет правила `replace / append / prepend / disable`;
- запускает `action` для разрешённых реакций.

---

## Чем отличается от `Endge.bind`

Важно не путать два модуля:

- `Endge.bind` - встраивает кастомный код в существующий `converter`, `action` или `runtime step`;
- `Endge.bindings` - исполняет сохранённые в домене `bindings`, связанные с event contracts.

То есть:

- `Endge.bind` = программная подмена реализации;
- `Endge.bindings` = декларативная реакция на событие.

---

## Из чего состоит связка

- `EventContract` - определяет допустимое событие;
- `Binding` - хранит реакцию;
- `EndgeBindingsResolver` - собирает итоговый набор;
- `Endge.bindings` - запускает соответствующие actions;
- `Endge.events` - отдельно уведомляет кодовых подписчиков.

---

## Как связан с `Raph`

`Endge.bindings` не подписывается на data-path напрямую.

Нижний слой здесь такой:

- `Raph` доводит изменение данных до runtime-host;
- runtime-host интерпретирует это как значимый факт;
- затем уже `Endge.bindings` решает, какие declarative реакции запускать.

Это важно, потому что `binding` работает не с грязными путями данных, а с каноническими событиями сущностей.

---

## Когда использовать

`Endge.bindings` нужен, когда реакцию хочет настраивать пользователь или конфигуратор.

Примеры:

- на `page.mounted` выполнить action;
- на `query.loaded` запустить сценарий;
- на `component.click` вызвать доменную реакцию.

Если же нужно вручную подменить код конкретного `converter` или `action`, это уже зона `Endge.bind`.
