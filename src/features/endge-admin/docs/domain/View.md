# View (Представление)

**Вид** - сущность домена, которая задаёт именованную конфигурацию экрана или набора данных. Вид связывает три сущности: компонент отображения, фильтр и запрос. Используется для переключения режимов отображения, списков и привязки к параметрам приложения.

## Что такое вид

Вид определяет «что показать пользователю» в данном режиме: какой компонент (таблицу, форму, DSL), с какими данными (запрос) и с какими условиями отбора (фильтр). У вида есть уникальный идентификатор (`identity`), отображаемое имя и описание.

## Связные сущности

Внутри вида задаются ссылки на три типа сущностей домена:

| Поле      | Описание |
|-----------|----------|
| **Компонент** | Компонент отображения (таблица или DSL). Определяет, как визуально отображаются данные. |
| **Фильтр**    | Фильтр данных. Задаёт условия отбора (поля, статические списки, словари и т.д.). |
| **Запрос**    | Запрос к данным (REST, GraphQL и др.). Определяет источник и форму данных. |

Все три поля опциональны: можно задать только компонент, только запрос и т.д. В редакторе вида сущности выбираются из списка или перетаскиванием из виджета «Домен»; рядом с каждым полем есть кнопка открытия выбранной сущности в новой вкладке.

Сущности с флагом «Унаследован от вида» могут хранить в `meta.inheritedFrom` ссылку на вид, от которого они унаследованы (например, сгенерированы генератором вида).
